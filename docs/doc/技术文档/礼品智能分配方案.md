# 槟界系统礼品智能分配方案技术文档

## 文档概述

本文档详细介绍了槟界系统中二维码礼品分配的智能算法方案，旨在为甲方提供清晰的技术实现说明和业务价值分析。

## 1. 背景与需求

### 1.1 业务背景
在槟界系统的营销活动中，需要为大量的二维码关联记录分配不同等级的礼品，确保：
- 每个配置的礼品都有实际的中奖机会
- 大奖（高价值礼品）能够均匀分布在不同包装中
- 避免因装袋顺序导致的中奖规律性
- 保证分配的公平性和随机性

### 1.2 核心需求
1. **公平性**：严格按照中奖率比例分配礼品
2. **随机性**：消费者无法预测中奖位置
3. **均匀性**：大奖不会集中在同一个包装中
4. **完整性**：每个配置的礼品至少分配1个

## 2. 技术方案概述

### 2.1 算法名称
**智能二维数组随机分配算法**

### 2.2 核心思想
通过二维数组和多重随机打散机制，实现礼品在包装层面的均匀分布和完全随机化。

### 2.3 技术特点
- **按列填充**：确保每个包装都包含不同位置的礼品
- **双重随机化**：先打散包装内部顺序，再打散包装间顺序
- **强制分配**：确保每个礼品至少分配1个
- **余数吸收**：最后一个礼品获得所有剩余数量

### 3.1 输入参数

/**

### 3.2 算法流程图

```
开始
  ↓
参数校验（marketingId是否为空）
  ↓
获取营销活动信息和礼品配置
  ↓
按父ID分组二维码记录
  ↓

  ↓

  ↓

  ↓

  ↓

```

### 3.3 核心步骤详解

#### 步骤1：参数校验
```java
if (marketingId == null) {
    // 清空所有礼品ID并返回
    qrCodeCorrelations.forEach(m -> m.setGiftId(null));
    return qrCodeCorrelations;
}
```

#### 步骤2：获取营销活动信息
```java
// 验证营销活动是否存在
TraceMarketing traceMarketing = traceMarketingMapper.selectById(marketingId);
if (traceMarketing == null) {
    throw new BaseException("营销活动不存在");
}

// 解析礼品配置（JSON格式）
List<GiftTmpDTO> marketingGiftDTOS = JSONUtil.toList(
    JSONUtil.toJsonStr(traceMarketing.getTrueGiftList()), 
    GiftTmpDTO.class
);
```

#### 步骤3：数据分组和维度计算
```java
// 按父ID分组（模拟包装结构）
List<List<QrCodeCorrelation>> parentId = CollUtil.groupByField(qrCodeCorrelations, "parentId");

// 计算数组维度
int size = qrCodeCorrelations.size();      // 总二维码数量
int parentIdSize = parentId.size();        // 父ID组数（中包数）
int packageNum = size / parentIdSize;      // 每组的包数
```

#### 步骤4：按中奖率分配礼品数量
```java
// 按中奖率排序（从高到低）
List<GiftTmpDTO> giftListSorted = marketingGiftDTOS.stream().sorted().toList();

// 创建礼品队列
ArrayDeque<Long> giftList = new ArrayDeque<>();
int suplusSize = size;  // 剩余数量

for (int i = 0; i < giftListSorted.size(); i++) {
    GiftTmpDTO marketingGift = giftListSorted.get(i);
    
    if (suplusSize == 0) break;
    
    if (i == giftListSorted.size() - 1) {
        // 最后一个礼品获得所有剩余数量
        Long[] longs = new Long[suplusSize];
        Arrays.fill(longs, marketingGift.getGiftId());
        giftList.addAll(new ArrayDeque<>(CollUtil.toList(longs)));
    } else {
        // 计算分配数量（强制最小值为1）
        BigDecimal winRate = marketingGift.getWinRate().divide(new BigDecimal(100), 4, RoundingMode.HALF_UP);
        int num = NumberUtil.mul(winRate, size).intValue();
        if (num < 1) num = 1;  // 关键：确保每个礼品至少分配1个
        
        Long[] longs = new Long[num];
        Arrays.fill(longs, marketingGift.getGiftId());
        giftList.addAll(new ArrayDeque<>(CollUtil.toList(longs)));
        suplusSize = suplusSize - num;
    }
}
```

#### 步骤5：二维数组按列填充
```java
// 创建二维数组：行=中包数，列=每组的包数
Long[][] arr = new Long[parentIdSize][packageNum];

// 按列填充礼品ID
for (int i = 0; i < size; i++) {
    int x = i % parentIdSize;  // 行索引（中包）
    int y = i / parentIdSize;  // 列索引（位置）
    arr[x][y] = giftList.poll();
}
```

#### 步骤6：双重随机打散
```java
// 第一步：对每个中包内的礼品进行随机打散
List<List<Long>> consumerGiftIds = new ArrayList<>();
for (int j = 0; j < parentIdSize; j++) {
    List<Long> c = Arrays.asList(arr[j]).subList(0, packageNum);
    Collections.shuffle(c);  // 打乱中包内部顺序
    consumerGiftIds.add(new ArrayList<>(c));
}

// 第二步：对所有中包的顺序进行再次随机打散
Collections.shuffle(consumerGiftIds);
```

#### 步骤7：最终分配
```java
// 合并所有礼品ID并分配给二维码记录
List<Long> consumerGiftIds0 = new ArrayList<>();
consumerGiftIds.forEach(consumerGiftIds0::addAll);
ArrayDeque<Long> arrayDeque0 = new ArrayDeque<>(consumerGiftIds0);

// 按顺序分配给二维码关联记录
qrCodeCorrelations.forEach(m -> {
    m.setGiftId(arrayDeque0.poll());
});
```

## 4. 业务价值分析

### 4.1 解决的核心问题

#### 问题1：大奖集中分布
**传统方案问题**：
```
中包1: [大奖,大奖,普通,普通,普通,普通]  // 集中2个大奖
中包2: [普通,普通,普通,普通,普通,普通]  // 无大奖
中包3: [大奖,大奖,普通,普通,普通,普通]  // 又集中2个大奖
```

**本方案解决方案**：
```
按列填充后：
列0: [大奖,普通,普通,大奖,普通,普通]
列1: [普通,大奖,普通,普通,大奖,普通]
列2: [普通,普通,大奖,普通,普通,大奖]
...
随机打散后：
中包1: [普通,大奖,普通,普通,大奖,普通]
中包2: [大奖,普通,普通,大奖,普通,普通]
中包3: [普通,普通,大奖,普通,普通,大奖]
```

#### 问题2：装袋顺序规律
**传统方案问题**：
- 消费者发现从中间开包总是能中奖
- 经销商可以通过特定进货批次获得更多大奖

**本方案解决方案**：
- **双重随机化**：先打乱包内部顺序，再打乱包间顺序
- **完全不可预测**：任何位置的中奖概率完全随机

### 4.2 核心优势

#### 1. 公平性保障
- **严格按比例**：严格按照中奖率分配礼品数量
- **强制分配**：每个礼品至少分配1个，确保兑现营销承诺
- **无偏分配**：算法本身不存在任何偏向性

#### 2. 用户体验提升
- **避免空包**：每个中包都有中奖机会
- **增加神秘感**：无法通过规律找到中奖方法
- **提升参与度**：随机性增加了消费者的参与乐趣

#### 3. 风险控制
- **法律合规**：避免虚假宣传风险
- **渠道管理**：防止经销商通过规律获取更多大奖
- **品牌保护**：维护品牌的公平性和诚信度

### 4.3 技术优势

#### 1. 算法稳定性
- **经过验证**：在实际业务中稳定运行
- **边界处理**：完善的异常处理机制
- **数据一致性**：确保分配结果的正确性

#### 2. 性能表现
- **时间复杂度**：O(n)，线性时间复杂度
- **空间复杂度**：O(n)，合理的内存使用
