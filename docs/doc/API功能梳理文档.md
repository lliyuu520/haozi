# API 功能梳理文档

## 概述

本系统 API 模块主要分为四个子模块，支持不同场景下的业务需求：
- **client**: 采集软件接口
- **in**: 内部微信小程序 API（经销商、区域经理、业务员、门店老板、二维码）
- **out**: 外部微信小程序 API（认证相关）
- **pda**: PDA设备专用 API（二维码绑定）

## 1. client 模块 - 采集软件接口

**Controller**: `ClientController`

### 功能清单
| 接口路径 | 方法 | 功能描述 | 权限要求 |
|---------|------|----------|----------|
| `/client/login` | POST | 采集软件登录 | @SaIgnore |
| `/client/traceProductList` | GET | 产品列表查询 | 需要登录 |
| `/client/pullQrCode` | POST | 拉取二维码（支持2级码、3级码） | 需要登录 + Redis分布式锁 |
| `/client/uploadQrCode` | POST | 上传二维码采集数据 | 需要登录 |
| `/client/collectRecord` | GET | 采集记录查询 | 需要登录 |
| `/client/traceOrderList` | GET | 订单列表查询（最近15天） | 需要登录 |

### 核心特性
- **登录认证**: 所有接口（除登录外）都需要在请求头携带 `Authorization: {token}`
- **分布式锁**: 二维码生成使用Redis锁防止并发冲突
- **数据限制**: 订单列表默认查询最近15天数据
- **分页支持**: 产品列表和订单列表支持分页查询

## 3. out 模块 - 外部微信小程序 API

### 3.1 WxMiniappAuthController - 微信小程序授权接口

**路径前缀**: `/out/wxMiniapp`

#### 功能清单
| 接口路径 | 方法 | 功能描述 |
|---------|------|----------|
| `/auth` | POST | 微信小程序授权认证 |

#### 核心特性
- **免认证**: `@SaIgnore` 注解，无需登录即可访问

- **用户管理**: 自动创建和更新小程序用户信息
- **安全认证**: 支持微信官方的安全认证机制

- **免认证**: `@SaIgnore` 注解，供微信官方回调使用

## 5. 公共组件和特性

### 5.1 认证机制
- **Sa-Token**: 系统统一使用Sa-Token进行认证管理
- **MaToken**: 微信小程序专用令牌认证机制
- **免认证接口**: 使用 `@SaIgnore` 注解标识

### 5.2 数据安全
- **Redis分布式锁**: 防止并发操作冲突
- **数据隔离**: 基于用户角色的数据权限控制
- **参数校验**: 所有接口都进行参数有效性验证

### 5.3 响应格式
- **统一返回**: 所有接口返回 `Result<T>` 格式
- **分页支持**: 列表查询支持 `PageVO<T>` 分页格式
- **错误处理**: 统一的异常处理和错误信息返回

### 5.4 业务特性
- **微信集成**: 深度集成微信小程序生态
- **二维码管理**: 完整的二维码生命周期管理
- **业务流程**: 覆盖订单、配送、核销等完整业务流程
- **数据统计**: 支持多维度的业务数据统计
